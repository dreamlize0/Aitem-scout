[PRD] AI 기반 영상 촬영 아이템 소싱 플랫폼 : AI템 스카우트 (AItem Scout)
1. 프로젝트 개요
•	프로젝트 이름: AI템 스카우트 (AItem Scout)
•	한 줄 설명: 영상 제작자를 위해 AI가 키워드, 테마, 타겟 맞춤형 촬영 아이템을 빠르게 발굴하고 제안하는 AI 기반 아이템 소싱 플랫폼
•	유형: 웹 서비스 (Web Service)
•	개발 난이도: 중상 (Medium-High)
2. 사용자 시나리오
•	누가 (Who)
•	유튜브 크리에이터, 방송 작가, 외주 영상 제작사 PD 및 기획자 등 '트렌디하고 조회수가 보장되는 촬영 소재'를 끊임없이 발굴해야 하는 영상 제작자.
•	언제 (When)
•	매주 정기 콘텐츠 기획 회의 전 아이템을 피칭해야 할 때.
•	새로운 채널 프로젝트(예: 해외 타겟 채널, 특정 연령대 타겟 브랜드 콘텐츠)를 론칭할 때.
•	기존 포맷의 아이템이 고갈되어 리프레시가 필요하거나, 급하게 대체 촬영지를 섭외해야 할 때.
•	왜 (Why)
•	포털 사이트 노가다 검색, 유튜브 피드 무한 스크롤, 타 방송 모니터링 등 아이템 서칭에 드는 물리적 시간을 획기적으로 줄이기 위함.
•	AI가 분석한 타겟 맞춤형 데이터(예: 40대 여성이 반응하는 데이트 코스, 일본인 관광객이 좋아하는 한국의 숨은 명소 등)를 기반으로 기획의 성공 확률을 높이고, 신속하게 콜드 연락 및 확정을 짓기 위함.
3. 핵심 기능 목록
[필수 기능 (Must-Have)]
	1.	AI 기반 통합 검색 엔진
•	키워드 검색: '맛집', '여행', '데이트', '골프' 등 직관적인 1차 카워드 입력 창 제공.
•	테마 검색: 1차 키워드를 뒷받침하고 맥락을 구체화하는 2차 필터링 기능 ('엄마랑 데이트', '가족과 함께', '가성비', '혼자만의 시간' 등).
•	시청 타겟별 검색: 성별, 연령별, 라이프스타일별 타겟 세그먼트 세부 설정 기능.
	2.	글로벌 확장(해외 타겟) 검색
•	'외국인 관광객용', '일본 시청자 타겟' 등 특정 국가 및 문화권을 타겟팅하면 AI가 해당 언어권의 트렌드와 검색 데이터를 반영하여 글로벌 맞춤 아이템 추천.
	3.	해외 트렌드 크로스 체크
•	구글 트렌드 및 글로벌 유튜브 Data API 등과 연동하여 선택한 아이템의 글로벌 예상 수요, 검색량 추이 및 트렌드 지표 시각화 리포트 제공.
	4.	아이템 추천 리포트 및 보관/공유
•	검색 조건에 매칭되는 촬영 후보지 및 아이템 리스트업 (추천 사유, 참고 레퍼런스, 기본 위치/연락처 포함).
•	선별한 촬영 아이템을 프로젝트별(예: '골프 채널 기획', '시니어 여행 브이로그') 폴더로 보관 및 관리.
•	외부 공유용 고유 링크(URL) 생성을 통해 로그인 없이도 팀원이나 클라이언트에게 웹뷰 형태로 공유 가능.
[선택 기능 (Nice-to-Have)]
	1.	AI 콜드 컨택(Cold Contact) 템플릿 지원
•	마음에 드는 촬영지/업체를 선택하면, AI가 해당 대상에게 발송할 섭외 메일, 인스타그램 DM, 제안서 초안을 자동으로 작성해 주는 기능.
4. 기술 스택
•	Front-end: 안티그래비티 (Antigravity) 기반 프론트엔드 빌드 및 UI 인터페이스 구성
•	Back-end & AI 빌더: Anthropic Claude (백엔드 비즈니스 로직 설계, 데이터 파이프라인 제어 및 프롬프트 엔지니어링)
•	AI & Search 스택 (★모듈형 플러그인 구조 설계)
•	핵심 원칙: 추후 새로운 플랫폼(예: TikTok, Pinterest 등)의 API나 신규 AI 모델이 출시되더라도 기존 아키텍처를 변경하지 않고 독립적인 모듈 형태로 즉시 플러그인(Plug-in) 장착이 가능한 모듈형 구조 적용.
•	Main LLM: 컨텍스트 분석, 추천 사유 요약 및 리포트 생성용 AI API (Claude 또는 GPT-4o)
•	글로벌 데이터: Google Trends API, YouTube Data API
•	국내 데이터: 네이버 검색 API (블로그/인사이트/쇼핑 트렌드), 카카오 검색 API (다음 웹/동영상)
•	SNS 트렌드 데이터: Meta Graph API (인스타그램 인기 해시태그 및 릴스 트렌드), X (Twitter) API v2 (실시간 트렌드), Threads API (라이프스타일 텍스트 트렌드)
•	방대 검색 보조: Tavily API 또는 Serper API (실시간 초고속 웹 서칭 및 데이터 정제용 엔진)
•	Database & Deployment: Supabase (사용자 인증, PostgreSQL 관계형 데이터베이스, 스토리지 및 서버리스 에지 펑션 배포 통합 운영)
5. 화면 구성
	1.	대시보드 / 메인 검색 홈 (Main Home)
•	화면 중앙: 직관적이고 직관적인 대형 키워드 검색창.
•	좌측/하단 필터 패널: 맥락 테마 선택, 시청 타겟 설정(성별/연령대/라이프스타일), 글로벌 타겟 국가(국내/일본/미국 등) 선택 옵션.
	2.	검색 결과 및 AI 추천 리포트 페이지 (Search Results & Report)
•	AI가 발굴한 촬영 아이템 카드 리스트 (좌측 스크롤 영역).
•	개별 아이템 카드 클릭 시 우측에 '상세 추천 리포트 레이어' 확장 오픈.
•	리포트 내 포함 사항: AI 추천 사유, 데이터 출처 원본 링크 아이콘, 국내외 트렌드 분석 차트, '보관함 저장' 및 '공유 링크 복사' 버튼.
	3.	내 보관함 / 프로젝트 폴더 페이지 (My Storage)
•	생성된 프로젝트 폴더별 큐레이션 리스트 뷰.
•	폴더별 팀원 초대 상태 관리 및 공유 링크 활성화/비활성화 토글 버튼.
	4.	설정 / 마이페이지 (Settings)
•	요금제 플랜 정보, 외부 API 연동 상태 점검, 워크스페이스 세팅 정보 관리.
6. 상세 기능 명세
① AI 통합 검색 및 출처 링킹 시스템
•	동작 방식: 사용자가 키워드, 테마, 타겟 조건을 조합해 검색을 실행하면, 백엔드 로직이 정의된 다중 포털 및 SNS API로 쿼리를 분산 호출합니다.
•	데이터 정제: 수집된 원본 데이터를 Main LLM에 정제 요청하여 타겟 시청층이 가장 매력적으로 느낄 만한 포인트 위주로 리포트를 구조화합니다.
•	출처 연동 (★중요): AI가 작성한 리포트 내의 모든 정보성 문장 및 레퍼런스에는 네이버, 유튜브, 인스타그램, X, 스레드 등의 오리지널 로고 아이콘이 배치됩니다. 사용자가 이를 클릭하면 새 창(Out-link)으로 실제 해당 원본 콘텐츠/포스팅으로 이동하여 데이터 신뢰도를 직접 검증할 수 있습니다.
② 글로벌 확장 및 트렌드 크로스 체크 모듈
•	동작 방식: '해외 타겟' 설정 시 AI는 수집 대상 키워드를 타겟 국가 언어(영어, 일본어 등)로 자동 번역하여 글로벌 트렌드 및 유튜브 API를 호출합니다.
•	시각화: 수집된 수치 데이터를 기반으로 '글로벌 트렌드 지수' 및 '관련 카테고리 상승률' 그래프를 대시보드 내에 실시간 렌더링합니다.
③ 프로젝트별 보관함 및 협업 공유 기능
•	동작 방식: 마음에 드는 아이템의 '저장' 버튼 클릭 시 Supabase DB에 해당 프로젝트 ID와 매핑되어 실시간 저장됩니다.
•	공유 아키텍처: 생성된 공유 링크는 난수화된 고유 토큰 URL 형태로 발급되어, 비로그인 상태의 서드파티 사용자도 해당 프로젝트의 아이템 리스트 및 AI 추천 리포트를 실시간으로 조회할 수 있는 웹뷰를 제공합니다.
7. 디자인 가이드
•	디자인 콘셉트: 영상 제작자를 위한 고몰입형 다크 테마 생산성 대시보드. (프리미어, 에프터이펙트 등 편집 툴과의 비주얼적 일치감 및 눈의 피로도 최소화)
•	컬러 시스템 (Color Palette)
•	배경색 (Background): 단일 다크 테마 (#121214, #1E1E22 등 다크 그레이/블랙 계열로 고정하여 불필요한 라이트모드 개발 공수 제거)
•	주 메인 컬러 (Primary): 스마트하고 시인성이 좋은 스카이 블루 (Sky Blue) 또는 일렉트릭 블루 (Electric Blue)
•	포인트 컬러 (Accent): 급상승 트렌드 지표 및 핵심 데이터 강조용 네온 그린 (Neon Green) 또는 비비드 오렌지 (Vivid Orange)
•	UI 컴포넌트 특이사항: 각 트렌드 카드 내에 네이버, 유튜브, 인스타그램, X, 스레드의 공식 브랜드 아이콘 컬러를 그대로 사용하여 시각적 직관성 확보.
•	타이포그래피: 텍스트 중심의 데이터 가독성을 확보하기 위해 자간과 행간이 최적화된 Pretendard 또는 Inter 폰트 적용.
8. 제약 사항
	1.	외부 플랫폼 API 쿼리 제한 (Rate Limit) 및 비용 관리
•	각 플랫폼(Meta, X, 네이버 등)의 API는 호출당 비용이 발생하거나 일일 제한이 엄격합니다. 동일 키워드 및 타겟 조합에 대한 반복 요청 시 외부 API를 재호출하지 않고, Supabase DB에 캐싱된 검색 결과를 일정 시간(예: 12~24시간) 동안 우선 반환하는 캐싱 로직이 필수적입니다.
	2.	실시간 데이터 스크래핑 및 LLM 분석 지연 (Latency)
•	다수의 API 및 SNS 데이터를 동시에 조회하고 AI가 요약 리포트를 생성하는 과정에서 약 3초~5초 내외의 네트워크 로딩 지연이 발생할 수 있습니다. 사용자 이탈을 막기 위해 검색 중임을 시각적으로 명확히 인지시키는 'AI 스카우팅 중...' 스켈레톤 UI 애니메이션 구현이 강제됩니다.
	3.	SNS 및 외부 API 정책 급변화 리스크
•	인스타그램, X 등의 정책 변경으로 특정 데이터 필드가 차단될 수 있습니다. 이를 대비해 검색 스택은 철저하게 '독립 모듈형 구조'를 유지하여 문제가 발생한 특정 API 모듈만 상시 차단/수정할 수 있도록 조치해야 합니다.
	4.	안티그래비티 및 클로드 AI 개발 환경 동기화
•	프론트엔드 빌더(안티그래비티)와 백엔드 로직 설계(클로드)의 유기적인 결합을 위해 초기 개발 플로우 구성 시 Supabase 인프라의 테이블 구조(스키마) 및 인증 API 명세서를 명확히 합의 및 동기화한 후 세부 기능 개발에 착수해야 합니다.
